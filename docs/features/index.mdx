## Open WebUI 的主要功能 ⭐

- 🚀 **轻松部署**：通过 Docker 或 Kubernetes（`kubectl`、`kustomize` 或 `helm`）无缝安装，支持包含 Ollama 的 `:ollama` 镜像和支持 CUDA 的 `:cuda` 镜像，带来无忧的使用体验。

- 🤝 **OpenAI API 集成**：轻松集成 OpenAI 兼容的 API，实现与 Ollama 模型的多样化对话。OpenAI API URL 可自定义，以便与各种第三方应用程序连接。

- 🛡️ **精细的权限和用户组管理**：管理员可以创建详细的用户角色和权限，确保用户环境安全。这种精细化不仅增强了安全性，还能为用户提供个性化体验，培养用户的主人翁意识和责任感。

- 📱 **响应式设计**：在台式电脑、笔记本和移动设备上享受流畅的使用体验。

- 📱 **移动端 PWA 应用**：在 `localhost` 或个人域名上享受离线访问的原生渐进式 Web 应用体验，界面流畅。为了确保 PWA 可以在您的设备上安装，必须在安全环境中提供服务，通常意味着必须通过 HTTPS 提供服务。
  - 要设置 PWA，您需要了解 Linux、Docker 和反向代理（如 `Nginx`、`Caddy` 或 `Traefik`）等技术。使用这些工具可以帮助简化构建和部署符合您需求的 PWA 的过程。虽然没有"一键安装"选项，且通过 HTTPS 安全部署 Open WebUI 实例需要一定的用户经验，但使用这些资源可以让您更轻松地创建和部署满足需求的 PWA。

- ✒️🔢 **完整的 Markdown 和 LaTeX 支持**：通过全面的 Markdown 和 LaTeX 支持提升大语言模型使用体验，实现更丰富的交互。

- 🧩 **模型构建器**：直接在 Open WebUI 中轻松创建 Ollama 模型。通过 [Open WebUI 社区](https://openwebui.com/) 集成，创建和添加自定义角色和智能体，自定义聊天元素，轻松导入模型。

- 📚 **本地和远程 RAG 集成**：通过我们在聊天中的检索增强生成（RAG）技术，探索文档的未来交互方式。可以将文档加载到工作区，然后使用 `#` 符号访问，或者在提示词前加上 `#` 后跟 URL 来集成网页内容。

- 🔍 **RAG 网页搜索**：您可以使用多种搜索提供商进行网页搜索，并将结果直接注入到本地 RAG 体验中。

- 🌐 **网页浏览功能**：通过在 `#` 命令后跟 URL，将网站无缝集成到聊天体验中。此功能可以将网页内容直接整合到对话中，从而增强交互的丰富性和深度。

- 🎨 **图像生成集成**：无缝集成图像生成功能，用动态视觉内容丰富您的聊天体验。

- ⚙️ **并发模型使用**：轻松同时使用多个模型，利用它们的独特优势获得最佳回应。并行利用多种模型形态来增强您的体验。

- 🔐 **基于角色的访问控制（RBAC）**：通过受限权限确保安全访问。只有授权人员可以访问您的 Ollama，而模型创建和拉取权限仅限管理员使用。

- 🌐🌍 **多语言支持**：通过国际化（`i18n`）支持，以您偏好的语言体验 Open WebUI。我们诚邀您加入扩展支持语言的行列！我们正在积极寻找贡献者！

- 🌟 **持续更新**：我们致力于通过定期更新、修复和新功能不断改进 Open WebUI。

### 🔧 管道支持

- 🔧 **管道框架**：通过我们的模块化插件框架（https://github.com/open-webui/pipelines）无缝集成和自定义您的 Open WebUI 体验，增强定制性和功能性。我们的框架允许轻松添加自定义逻辑和集成 Python 库，从 AI 智能体到家庭自动化 API。

- 📥 **上传管道**：可以直接从`管理面板` > `设置` > `管道`菜单上传管道，简化管道管理流程。

#### 我们的管道框架潜力无限，几乎没有边界。从一些预构建的管道开始，帮助您入门！

- 🔗 **函数调用**：通过管道无缝集成[函数调用](https://github.com/open-webui/pipelines/blob/main/examples/filters/function_calling_filter_pipeline.py)，使用高级函数调用功能增强您的大语言模型交互。

- 📚 **自定义 RAG**：无缝集成[自定义检索增强生成（RAG）](https://github.com/open-webui/pipelines/tree/main/examples/pipelines/rag)管道，通过自定义 RAG 逻辑增强您的大语言模型交互。

- 📊 **使用 Langfuse 监控消息**：通过 [Langfuse](https://github.com/open-webui/pipelines/blob/main/examples/filters/langfuse_filter_pipeline.py) 管道实时监控和分析消息交互的使用统计。

- ⚖️ **用户速率限制**：通过控制发送到大语言模型的请求流量，使用[速率限制](https://github.com/open-webui/pipelines/blob/main/examples/filters/rate_limit_filter_pipeline.py)管道有效管理 API 使用，防止超出速率限制。

- 🌍 **LibreTranslate 实时翻译**：使用 [LibreTranslate](https://github.com/open-webui/pipelines/blob/main/examples/filters/libretranslate_filter_pipeline.py) 管道将实时翻译集成到您的大语言模型交互中，实现跨语言交流。
  - 请注意，此管道需要在 Docker 容器中进一步设置 LibreTranslate 才能工作。

- 🛡️ **有害消息过滤**：我们的 [Detoxify](https://github.com/open-webui/pipelines/blob/main/examples/filters/detoxify_filter_pipeline.py) 管道自动过滤有害消息，维护干净安全的聊天环境。

- 🔒 **LLM-Guard**：使用 [LLM-Guard](https://github.com/open-webui/pipelines/blob/main/examples/filters/llmguard_prompt_injection_filter_pipeline.py) 管道确保大语言模型交互的安全性，其提示词注入扫描器可以检测和缓解针对大语言模型的巧妙输入操作。这可以保护您的大语言模型免受数据泄露，并增加对提示词注入攻击的抵抗力。

- 🕒 **对话轮次限制**：通过[对话轮次限制](https://github.com/open-webui/pipelines/blob/main/examples/filters/conversation_turn_limit_filter.py)管道设置对话轮次限制，改进交互管理。

- 📈 **OpenAI 生成统计**：我们的 [OpenAI](https://github.com/open-webui/pipelines/blob/main/examples/pipelines/providers/openai_manifold_pipeline.py) 管道为 OpenAI 模型提供详细的生成统计信息。

- 🚀 **多模型支持**：我们与[各种提供商](https://github.com/open-webui/pipelines/tree/main/examples/pipelines/providers)的 AI 模型无缝集成，让您可以选择和交互的语言模型范围更广。

#### 除了丰富的功能和自定义选项外，我们还提供[一个可直接使用的示例管道库](https://github.com/open-webui/pipelines/tree/main/examples)以及[一个实用的示例脚手架管道](https://github.com/open-webui/pipelines/blob/main/examples/scaffolds/example_pipeline_scaffold.py)来帮助您入门。这些资源将简化您的开发过程，让您能够使用管道和 Python 快速创建强大的大语言模型交互。祝您编码愉快！💡

### 🖥️ 用户体验

- 🖥️ **直观的界面**：聊天界面采用以用户为中心的设计理念，参考了 ChatGPT 的优秀用户界面设计。

- ⚡ **快速响应**：提供稳定可靠的快速响应体验。

- 🎨 **启动画面**：简洁的加载启动画面，带来更流畅的用户体验。

- 📦 **Pip 安装方式**：支持通过 `pip install open-webui` 命令安装 Open WebUI，大大简化了安装流程，特别适合新用户。详情请访问：https://pypi.org/project/open-webui/。

- 🌈 **主题定制**：提供丰富的个性化选项，包括多款简洁优雅的主题、自定义聊天背景图片，以及浅色、深色和 OLED 深色三种显示模式 - 您也可以让系统智能选择！

- 💻 **代码语法高亮**：内置的语法高亮功能提升代码可读性，呈现清晰整洁的代码视图。

- ↕️ **双向聊天支持**：支持在从左到右和从右到左的聊天方向之间自由切换，完美适配不同语言的书写习惯。

- 📱 **移动端手势操作**：在移动设备上通过简单的滑动手势即可控制侧边栏的显示与隐藏。

- 📂 **统一工作区**：集成的工作区将所有模型文件、提示词、文档、工具和函数集中在一处，有效简化工作流程。

- 💾 **设置持久化**：所有设置都会自动保存在 config.json 文件中，确保您的偏好设置随时可用。

- ❓ **便捷文档访问**：在大屏设备（如台式电脑和笔记本）上，主界面右下角的问号按钮提供快速访问 Open WebUI 文档和键盘快捷键的入口。

- 📜 **更新管理**：在`设置` > `关于` > `查看最新动态`菜单中，您可以查看完整的更新日志，了解最新功能、改进和问题修复，并检查是否有新版本可用。

### 💬 对话功能

- 🔍 **RAG 嵌入支持**：可以直接在`管理面板` > `设置` > `文档`菜单中更改检索增强生成（RAG）的嵌入模型，增强文档处理能力。此功能支持 Ollama 和 OpenAI 模型。

- 📜 **RAG 引用功能**：检索增强生成（RAG）功能支持通过引用点追踪提供给大语言模型的文档上下文，使内容来源更加透明。

- 🌟 **增强型 RAG 管道**：为 RAG 嵌入功能提供可切换的混合搜索功能，通过 `BM25` 算法增强检索能力，使用 `CrossEncoder` 进行结果重排序，并支持自定义相关性分数阈值。

- 📹 **YouTube RAG 管道**：专门的 YouTube 视频处理管道，可通过视频 URL 直接处理视频内容，实现与视频转录文本的无缝交互和总结。

- 🔄 **多模态支持**：支持与多模态模型（如 `LLaVA`）的交互，实现包括图像处理在内的多种模态操作。

- 🤖 **多模型支持**：支持在不同模型之间快速切换，实现灵活多样的对话交互。

- 👥 **'@' 模型调用**：在对话中使用 `@` 命令指定模型名称，即可无缝切换到任何可用的本地或外部模型，充分利用多个模型的协同能力。

- 🏷️ **对话标签管理**：通过标签系统轻松组织和检索聊天记录，便于快速查找和数据整理。

- 👶 **聊天克隆功能**：支持一键克隆和保存任何对话的快照，方便后续参考或继续对话。只需点击聊天下拉选项中的`克隆`按钮，即可创建对话副本，实现便捷的会话管理和分享。

- 📜 **预设提示词系统**：在聊天输入框中使用 `/` 命令快速调用自定义预设提示词。支持加载预定义的对话模板，提高交互效率。可通过 [Open WebUI 社区](https://openwebui.com/) 导入提示词，也可以创建个性化提示词。

- 📅 **提示词变量系统**：支持在系统提示词或通过斜杠命令选择提示词时使用动态变量，包括：
  - `{{CLIPBOARD}}` - 剪贴板内容
  - `{{CURRENT_DATE}}` - 当前日期
  - `{{CURRENT_DATETIME}}` - 当前日期时间
  - `{{CURRENT_TIME}}` - 当前时间
  - `{{CURRENT_TIMEZONE}}` - 当前时区
  - `{{CURRENT_WEEKDAY}}` - 当前星期
  - `{{USER_NAME}}` - 用户名
  - `{{USER_LANGUAGE}}` - 用户语言
  - `{{USER_LOCATION}}` - 用户位置（需要 HTTPS 连接并在`设置` > `界面`中启用）
  注意：使用 `{{CLIPBOARD}}` 变量需要获取剪贴板访问权限。

- 🧠 **记忆管理**：通过`设置` > `个性化` > `记忆`菜单管理大语言模型的记忆内容。支持添加、编辑和删除记忆条目，实现个性化的上下文管理。

### 💻 模型管理

- 🛠️ **模型构建器**：在模型工作区中提供持久化的模型构建和编辑环境，支持所有模型的配置管理。

- 📚 **模型知识库**：支持直接在模型工作区为每个模型关联函数和文档，丰富模型的知识储备。

- 🗂️ **模型预设管理**：支持创建和管理 Ollama 和 OpenAI API 的模型预设配置。

- 🏷️ **模型标签系统**：在模型工作区中使用标签系统进行模型分类和组织。

- 📋 **模型选择器排序**：通过直观的拖放操作在模型工作区中调整模型顺序，更改会自动同步到模型选择下拉菜单。

- 🔍 **智能模型选择器**：配备搜索过滤功能的模型选择下拉菜单，展示包含标签和描述在内的详细模型信息。

- ⚙️ **高级参数调优**：提供对模型参数的精细控制，包括：
  - `seed`（随机种子）
  - `temperature`（温度）
  - `frequency penalty`（频率惩罚）
  - `context length`（上下文长度）
  等关键参数的调整。

- 🔄 **快速模型导入**：支持从 [Ollama 模型库](https://ollama.com/library/) 直接复制 `ollama run {model:tag}` 命令到模型选择器，实现一键选择和拉取模型。

- 🗂️ **Modelfile 创建**：通过 `管理面板` > `设置` > `模型` > `创建模型` 快速创建 Ollama 模型文件。

- ⬆️ **GGUF 模型支持**：在 `管理设置` > `设置` > `模型` > `实验性功能` 中提供 GGUF 文件上传功能，支持：
  - 本地文件上传
  - Hugging Face 模型下载
  实现 Ollama 模型的便捷创建。

- ⚙️ **默认模型配置**：
  - 移动端：通过 `设置` > `界面` 设置新对话默认模型
  - 桌面端：在新建对话时通过模型选择器直接设置

- 💡 **响应分析**：提供每次模型响应的详细分析，包括：
  - 外部 API 调用信息
  - 本地模型运行数据

- 📥🗑️ **模型管理工具**：提供模型的一键下载和删除功能。

- 🔄 **批量更新**：支持一键更新所有已安装的 Ollama 模型。

- 🍻 **角色卡集成**：支持在模型构建器中集成 TavernAI 角色卡 PNG 文件，增强模型的个性化和互动体验。

- 🎲 **模型实验室**：提供测试版模型游乐场环境，支持在沙盒中安全测试和调优模型参数，确保部署前的功能验证。

### 👥 协作功能

- 🗨️ **本地聊天分享**：提供高效的聊天链接生成和分享机制，支持用户间的便捷协作与交流。

- 👍👎 **人类反馈标注**：支持通过点赞或点踩对对话内容进行评分，并提供文字反馈，用于构建人类反馈强化学习（RLHF）数据集。您的反馈可用于模型训练和微调，同时确保数据的本地存储和隐私保护。

- 🤝 **社区互动**：通过`分享到 Open WebUI 社区`功能，将精选对话分享到 [Open WebUI 社区](https://openwebui.com/)，实现用户间的知识共享和协作。
  - 使用说明：
    - 需要登录 Open WebUI 社区账户
    - 分享有助于建设活跃社区
    - 促进知识传播和问题解决
    - 此功能为可选项
    - 管理员可在`管理设置` > `设置` > `常规`中控制此功能的开启或关闭

### 📚 历史记录与归档

- 📜 **对话历史**：
  - 通过导航侧边栏轻松管理对话记录
  - 可在`设置` > `对话`中关闭历史记录功能
  - 支持选择性记录新对话

- 🔄 **重生成记录**：完整保存并展示模型响应的所有重新生成历史，方便追踪和复查。

- 📬 **对话归档**：
  - 支持将已完成的对话存档
  - 便于后续查阅和继续对话
  - 保持主界面整洁有序

- 🗃️ **批量归档**：提供一键归档所有对话的功能，实现高效的对话管理。

- 📦 **归档导出**：支持将所有归档对话导出为单个 JSON 文件，便于数据备份和迁移。

- 📄 **多格式导出**：支持将单个对话导出为多种格式：
  - `.json` 格式（完整数据）
  - `.pdf` 格式（便于阅读）
  - `.txt` 格式（纯文本）

- 📤📥 **数据迁移**：
  - 导入功能：支持批量导入历史对话
  - 导出功能：支持选择性导出对话记录
  确保对话数据的便捷迁移和备份。

- 🗑️ **清理功能**：提供一键删除所有对话的选项，支持彻底清理并重新开始。

### 🎙️ 语音与无障碍

- 🗣️ **语音输入**：

  - 增强交互的情感体验

- 🎙️ **免提对话**：
  - 支持无需手动操作的语音对话
  - 提供更便捷的交互方式
  注意：需要 HTTPS 安全连接。

- 📹 **视频对话**：
  - 支持与视觉模型（如 LlaVA、GPT-4）的视频交互
  - 提供多模态通信体验
  注意：需要通过 HTTPS 安全连接启用摄像头和麦克风。

- 👆 **触控中断**：
  - 支持在移动设备上通过点击停止语音输出
  - 提供精确的交互控制

- 🔊 **语音合成配置**：
  - 支持自定义 OpenAI 兼容的文本转语音接口
  - 可定制模型响应的语音朗读效果
