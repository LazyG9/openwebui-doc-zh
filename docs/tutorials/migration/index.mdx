## 从内部 LiteLLM 迁移到外部 LiteLLM

Open WebUI 的早期版本（0.2 之前）在内部运行 `litellm`。为了提高模块化和灵活性，我们建议在独立的容器中运行 `litellm`。本指南将指导您完成迁移过程。

### 1. 下载您的 `config.yaml` 文件

在进行任何更改之前，从 Open WebUI 管理设置窗口下载您现有的 `config.yaml` 文件。此文件包含您当前的 `litellm` 配置。

![LiteLLM config.yaml 下载](/img/migration_litellm_config.png)

### 2. 运行独立的 `litellm` 容器

使用以下 `docker run` 命令启动专用的 `litellm` 容器：

```bash
docker run -d \
  -p 4000:4000 \
  --name litellm \
  -v ./config.yaml:/app/config.yaml \
  -e LITELLM_MASTER_KEY=your_secret_key \
  --restart always \
  ghcr.io/berriai/litellm:main-latest \
  --config /app/config.yaml --port 4000
```

- 将 `./config.yaml` 替换为已下载的 `config.yaml` 文件的实际路径。
- 为 `LITELLM_MASTER_KEY` 设置一个安全的 API 密钥。这确保了对您的 `litellm` 实例的受控访问。

### 3. 将 `litellm` 连接到 Open WebUI

一旦 `litellm` 容器启动并运行：

1. 进入 Open WebUI 设置。
2. 在"连接"下，添加一个新的"OpenAI"连接。
3. 将基础 URL 设置为 `http://host.docker.internal:4000/v1`（如有必要，调整端口）。
4. 对于 API 密钥，使用您在步骤 2 中定义的 `LITELLM_MASTER_KEY` 值（例如，`your_secret_key`）。

恭喜！您已成功迁移到外部 `litellm` 设置，提高了 Open WebUI 安装的灵活性和可维护性。

## 将内容从 Ollama WebUI 迁移到 Open WebUI

由于最近从 [Ollama WebUI 更名为 Open WebUI](https://github.com/open-webui/open-webui/discussions/602)，docker 镜像已被重命名。对于之前使用 Ollama WebUI 并想开始使用新镜像的用户，需要执行额外的步骤。

## 更新到 Open WebUI 而不保留数据

如果您想更新到新镜像但不想保留任何以前的数据（如对话、提示、文档等），可以执行以下步骤：

:::danger

执行这些步骤将删除所有当前配置选项、聊天历史等。只有 LLM 模型会被保留

:::

```bash
docker rm -f ollama-webui
docker pull ghcr.io/open-webui/open-webui:main
[插入您使用新 Docker 镜像名称安装时使用的等效命令]
docker volume rm ollama-webui
```

例如，对于本地安装，命令是 `docker run -d -p 3000:8080 --add-host=host.docker.internal:host-gateway -v open-webui:/app/backend/data --name open-webui --restart always ghcr.io/open-webui/open-webui:main`。有关其他安装命令，请查看本 README 文档的相关部分。

## 更新到 Open WebUI 并保留数据

如果您想更新到新镜像并迁移所有以前的设置（如对话、提示、文档等），可以执行以下步骤：

```bash
docker rm -f ollama-webui
docker pull ghcr.io/open-webui/open-webui:main
# 创建新卷并使用临时容器从一个卷复制到另一个卷，参见 https://github.com/moby/moby/issues/31154#issuecomment-360531460
docker volume create --name open-webui
docker run --rm -v ollama-webui:/from -v open-webui:/to alpine ash -c "cd /from ; cp -av . /to"
[插入您使用新 Docker 镜像名称安装时使用的等效命令]
```

一旦您验证所有数据都已迁移，就可以使用以下命令删除旧卷：

```bash
docker volume rm ollama-webui
```

### 从本地 Git 仓库迁移

如果您是从 git 安装的，在项目目录中使用了 `docker compose up`，您的卷名将以文件夹名为前缀。
因此，如果您的 OpenWebUI 路径是：`/home/myserver/ollama-webui/`，卷名将是 "ollama-webui_open-webui" 和 "ollama-webui_ollama"。

要将内容复制到常规的 docker 安装中，您可以运行类似的迁移命令。在我们的特定情况下，命令将是：
```bash
docker rm -f open-webui
docker rm -f ollama
docker pull ghcr.io/open-webui/open-webui:main
docker pull ghcr.io/open-webui/open-webui:ollama
docker volume create --name open-webui
docker volume create --name ollama
docker run --rm -v ollama-webui_open-webui:/from -v open-webui:/to alpine ash -c "cd /from ; cp -av . /to"
docker run --rm -v ollama-webui_ollama:/from -v ollama:/to alpine ash -c "cd /from ; cp -av . /to"
```

根据您是否已安装 ollama，或者是否已经有相同的卷名，某些命令**可能会抛出错误**，但通常可以安全地忽略，因为我们正在覆盖。

然后，按照入门指南中的描述正常启动两个容器。

一旦您验证所有数据都已迁移，就可以使用上面提到的 `docker volume rm` 命令删除旧卷。
